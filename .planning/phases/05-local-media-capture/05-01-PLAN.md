<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ↔ user | Permission prompts and capture consent are OS/browser controlled; app must not assert trust beyond user intent. |
| Client code ↔ logs | Avoid logging stream IDs, tracks, or raw buffers that could aid fingerprinting in shared logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Information disclosure | `use-media-capture.ts` | mitigate | Do not `console.log` `MediaStream`, track IDs, or `getSettings()` in default builds; keep any debug behind `import.meta.env?.DEV` or remove before commit. |
| T-5-02 | Denial of service | `getUserMedia` retry loops | mitigate | Single user-triggered start per click; no tight retry loop on failure. |
| T-5-03 | Tampering | Mocked tests | accept | Test doubles only in `*.test.ts`; no impact on production bundle. |
</threat_model>
