## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input→client form model | Untrusted values enter create/edit fields before API calls |
| client API wrapper→server routes | Mutation requests may expose/alter location lifecycle state |
| hub view state boundary | Include-archived and error rendering decide what sensitive data is shown |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | T | form submission payloads | mitigate | Build payload only from whitelist keys (`name`, `code`, `notes`) and validate required name before submit |
| T-08-07 | I | error UI in hub/detail | mitigate | Render generic access-denied/request-failed copy; never display inaccessible location ids from error strings |
| T-08-08 | S | actor context in client calls | accept | Client forwards `x-user-id` dev actor header; server remains trust boundary enforcing authorization |
| T-08-09 | T | archive/restore action triggers | mitigate | Require explicit confirmation for archive and call dedicated restore endpoint; do not perform local-only lifecycle toggles |
</threat_model>
