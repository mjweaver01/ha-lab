## Interaction Contracts

- Locations hub uses a dense sortable table layout with columns: name, status, updated timestamp, optional code/slug, and row actions.
- Both row click and explicit `Open` action must navigate to location detail; both paths are equivalent.
- Default view includes only active locations; `Include archived` toggle is off by default and persists for the current page session.
- Create and edit use one shared form model on a dedicated detail/edit surface (no inline row edit mode).
- `name` is required; `code/slug` and `notes/description` are optional. Show inline field errors and a request-level error banner on submit failure.
- Archive is soft-only (`archived_at` lifecycle). Archive action requires confirmation; restore action is available in hub when archived entries are visible.
- State contracts:
  - Loading: show panel-level loading message while preserving page layout.
  - Empty active list: render empty-state panel with CTA to create the first location.
  - Error: render non-leaky error banner and retain existing list content when possible.
- Access-denied responses must be explicit but non-revealing; UI copy must not disclose inaccessible location identifiers.
